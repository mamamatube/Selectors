基本セレクター

名称 内容
要素セレクター
(A{ })		 <h1>、<p>などの要素を指定するためのセレクター
全称セレクター
(*{ })		 *{ }を使い、全ての要素に対し指定するセレクター。
box-sizingの設定によく使われる。
クラスセレクター
(.A{ })		 class属性につけた値の先頭に.をつけ指定するセレクター。
属性の値には_や-の使用はできるが、数字から始まってはいけないなど、ルールがある。
また、1つの要素に対し、空白で区切って複数の値を指定できる。
idセレクター
(#A{ })		 class属性と同じように、要素にid属性をつけ、値の先頭に#をつけ指定するセレクター。
classと異なり複数の値を設定できない。
また、同じページ内で複数の要素に対し同じidの値は設定できないルールがある。
(JavavScriptでよく使われる)
属性セレクター
([A]{ })		 htmlで設定した特定の属性がついた要素のみに対してCSSを指定したい場合に用いる。
指定したい属性を[ ]で括り指定する。
例)targetがついた要素にだけmarginを指定したい場合→[target]{ margin:~~ }
擬似クラス
(:A{ })
特定の要素ではなく、「形式的な位置」にプロパティを指定したい場合などに用いるセレクター。
例)兄弟要素のうち、"常に3番目にある要素"にスタイルを指定したい場合は
:nth-child(3){ }と指定する。
この時、()の中に指定できる値として、奇数＝odd、偶数even、3倍数目＝3nなどの種類がある。
また、最初の要素を指す:first-childや最後の要素:last-childもある

ユーザーの操作によって状態が変わる要素には、それぞれ異なる擬似クラスも使用できる
input要素、button要素などには
マウスホバーした時にスタイルが変わるよう:hoverを使用したり
input要素のような選択した状態(focus)が持てる要素には:focus、
buttonのように押し込んだり、クリックした時の状態を表現する際には:activeを使用したりする

否定の擬似クラス:not()をつけると、()内に指定した要素を除外した要素が選択されるが、
複合セレクターなどで範囲を限定しないとページ全体に影響が及ぶ場合がある為注意が必要。
擬似要素
(::A{ })		 CSSだけで擬似的な要素を追加することができるセレクター。
よく使われる::before、::afterを例に、HTMLにp要素があった場合
複合セレクターを用いてp::before{ content:'-';}、p::after{ content:'-';}と置くと、
CSSから、HTMLのp要素の前後に対し-を付与することができる。
なお、あくまで指定した要素内の前後に対して挿入されるセレクターであり、
タグの前後に追加されるわけではない点に注意。

複合セレクター
(AB{ })

基本セレクターを2つ以上繋げたセレクター。
※Aには「要素セレクター」or「全称セレクター」のみ指定でき、
その後ろ(B以降）には基本セレクターをどれでも繋げることが可能。
例)ページ内に<div>やclass="box"が複数ある中、 < Box</>のみを指定したい場合
→{ }と指定する

セレクターリスト
(A,B{ })

複数の要素に対して同じプロパティ・値を指定したい時、複数の要素を,で区切ってまとめて指定することができる。
例)<h1>と<p>それぞれにbackground:pink;を指定したい場合
↓(※Google社のスタイルガイドではセレクターリストはカンマごとに改行が推奨)
h1,
p{ background:pink; }でOK

結合子

名称 内容
子結合子
(A > B{ })		 ある要素の中の”特定の子要素”のみに対してプロパティを指定したい場合に用いるセレクター。
例）<section>要素内の<p>にだけcolor:red;を指定したい
→section > p{ color:red; }
子孫結合子
(A B{ })		 ある要素の中の特定の子要素”+孫要素以降”に対してまで
プロパティを指定したい場合に用いるセレクター。
例）<section>要素内の<p>、さらにその中の要素（<section>視点の孫要素）にまでcolor:red;を指定
→section p{ color:red; }
隣接兄弟結合子
(A + B{ })		 複数要素が並んでいる際、特定の並びを持つ要素のみを選択するセレクター
A + B{ }の場合、「直前の兄弟要素に<A>を持つするセレクター」となる

セレクターの優先度

同一の要素に対して同じセレクター・値を指定した場合には、セレクターの優先度によって反映される値が変わる。
①idセレクターが現れる回数
②クラスセレクター、擬似クラス、属性差レクターが現れる回数
③要素セレクター、擬似要素が現れる回数
上記①→③の勝ち抜き方式で優先度が決定する。（開発者ツールで確認するのが一般的）
なお、さらに優先度が高い方法として
・HTMLの要素に直接スタイルをつける方法
・プロパティの値の最後に!importantをつける方法（直接スタイルつけるより最優先）もある
